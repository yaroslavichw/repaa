# Скрипт синхронизации локальных репозиториев

## Описание
Проект содержит два скрипта (`sync.sh` для Unix-систем и `sync.ps1` для Windows) предназначенных для автоматической синхронизации выбранного локального репозитория с его удалённой копией и последующего копирования обновленных файлов в другой локальный репозиторий. 

Скрипты разработаны с учётом требований к проверке путей и корректной обработке скрытых директорий (исключение папки `.git`). Также учтена разница в обработке путей и регистров между Windows и Linux/macOS.

## Реализованный функционал
1. **Проверка путей:** Скрипты проверяют наличие исходной и целевой директорий перед началом работы, а также удостоверяются, что исходная папка является валидным Git-репозиторием.
2. **Синхронизация (`git pull`):** Автоматически подтягиваются последние изменения из удалённого репозитория (`origin/master`).
3. **Безопасное копирование:** * В Bash-версии используется `rsync`.
   * В PowerShell-версии используется `Copy-Item`.
   Папка `.git` строго исключается, чтобы не перезаписать историю целевого репозитория. Существующие файлы заменяются автоматически.
4. **Визуальный контроль:** Скрипты последовательно выводят этапы выполнения в терминал и ожидают нажатия клавиши перед закрытием.

## Системные требования
* **Для `sync.sh`:** Linux (Debian/Ubuntu), macOS или WSL. Пакеты `git` и `rsync`.
* **Для `sync.ps1`:** Windows 10/11, PowerShell 5.1+, установленный `git`.

## Инструкция по запуску

### Для Linux / macOS (Bash)
1. Откройте терминал.
2. Сделайте скрипт исполняемым:
   ```bash
   chmod +x sync.sh
   ```
3. Запустите скрипт:
   ```bash
   ./sync.sh
   ```

### Для Windows (PowerShell)
1. Откройте PowerShell.
2. Разрешите выполнение локальных скриптов (если не разрешено ранее):
   ```powershell
   Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
   ```
3. Запустите скрипт:
   ```powershell
   .\sync.ps1
   ```
