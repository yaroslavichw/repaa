## Скрипт синхронизации репозиториев

Автоматизация рутинных задач с помощью скриптов - важная часть профессиональной деятельности любого разработчика!

> При создании скриптов следует учитывать особенность, что две одинаковые буквы английского алфавита, написанные в разных регистрах (верхний или нижний) - для **Windows** будут неразличимы, а для **Unux** (Linux и MacOS) - разными символами! Например: 'a' и 'A' - для винды это одни и теже буквы, а для **Linux/MacOS** - разные. Папки `Content` и `сontent` для винды и линукса обрабатываются по-разному.

> Будьте внимательны к генерируемому нейронкой коду, проверяйте пути. Есть риск потерять свои данные из-за некорретный путей при копировании из одной папки в другую!

- **Создать** при помощи нейросети скрипт на **bash** и **powershell**, который синхронизирует выбранный локальный репозиторий с его удалённой копией (команда `git pull`), <u>скопирует</u> его содержимое с проверкой пути, кроме папки `.git`, в локальную копию другого репозитория с проверкой пути и с заменой имеющихся там файлов и папок с теми же именами.
- Опционально можно добавить в срипт ещё и команду `git push` в локальной копии репозитория, в котором все обновления нужно запушить;
- Опционально в вашем скрипте можно добавить пропуск или обработку приватных репозиториев и т.п.;
- Опционально можно сделать так, чтобы ваш скрипт выполнялся явно, в окне автоматически запущенного Терминала, для визуального контроля его выполнения;
- **Протестировать** скрипт можно внутри Docker-контейнера с **Ubuntu** или **WSL**, виртуальной **Windows** (в **Virtual Box** или **Hyper V**). Для тестирования в указанных ОС придётся в них склонировать два репозитория - мой и ваш, отредактировать пути в скрипте и запустить его для выполнения синхронизации. Если синхронизация прошла успешно, то можно поместить его в автозагрузку вашей хостовой ОС для удобства. Перед тестированием вашего скрипта в виртуальных ОС не забудьте установить в них **git**!
- **Результат** (код скрипта с описанием его работы) прислать в виде ссылки на файл `README.md` с описанием проекта и файлов самого скрипта для `bash` и `powershell`
